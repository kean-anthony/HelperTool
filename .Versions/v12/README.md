## Overview

Version 12 introduces a new automated file seeding system and major improvements to the Secret Holder search engine.

This update focuses on:

- Automated folder-based file generation

The goal of this version is to reduce repetitive setup work while improving secret management workflows across larger projects and repositories.

# New Features

## File Seeder System (12.0.0)

Version 12 introduces a new **File Seeder System**.

The feature allows users to automatically generate multiple files inside a selected folder using a simple seeding workflow.

Users can now:

- Select a target folder
- Input multiple file names
- Automatically generate files in bulk
- Seed project structures faster
- Reduce repetitive manual file creation

The seeder system is designed to improve project initialization workflows and speed up repository setup operations.

Supported use cases include:

- Creating module structures
- Generating component files
- Initializing feature folders
- Creating utility templates
- Rapid workspace scaffolding
- Bulk repository preparation

The workflow is intentionally lightweight and optimized for fast repetitive development tasks.

Example workflow:

1. Select target folder
2. Enter file names
3. Run seeder
4. Files are automatically created inside the selected directory

This significantly reduces repetitive manual file creation during development.

# Improvements

## Secret Holder Search Engine Improvements (12.0.1)

The Secret Holder system has received major search improvements.

The updated search engine now provides:

- Faster search performance
- Improved result filtering
- Better keyword matching
- Smarter secret lookup behavior
- Improved large-list handling
- More responsive search updates

Search accuracy has also been improved to better detect partial matches and related search terms.

The new implementation is optimized for repositories and environments containing large amounts of stored secrets, tokens, credentials, and configuration entries.

Key improvements include:

- Reduced search latency
- Better search responsiveness
- Improved lookup consistency
- Faster filtering across large datasets
- Cleaner result matching behavior

These improvements make secret retrieval significantly faster and more reliable during active development workflows.
